# 02 — Especificação de Frontend: Interface do Cliente

Escopo: `cliente_app.py`. Depende de `backend.py`, `aiengine.py` e
`theme.py`.

## Objetivo da tela
Permitir que qualquer usuário SAP abra um chamado de suporte e acompanhe o
andamento/solução dos seus próprios chamados, sem enxergar chamados de
outras pessoas.

## Execução
```bash
python -m streamlit run cliente_app.py --server.port 8501
```
Config da página: `page_title="Portal do Cliente - Helpdesk SAP"`,
`page_icon="🎫"`, `layout="wide"`.

## Tema
Aplicado via `theme.aplicar_tema_dark()` — fundo `#0d1117`, texto `#c9d1d9`,
sidebar `#161b22`, cards de métrica com borda `#30363d`. Qualquer novo
componente visual criado nesta tela deve reaproveitar essas variáveis de
cor (não hardcodar novas cores sem necessidade).

## 1. Sidebar — Identificação

| Elemento | Tipo | Comportamento |
|---|---|---|
| "Seu usuário/nome" | `st.text_input` | Valor persistido em `st.session_state["usuario_atual"]`. Texto de ajuda deixa claro que **não é login real**, é só identificação para filtrar os próprios chamados. |

**Regra de bloqueio**: se o campo estiver vazio, a tela inteira mostra um
`st.warning` pedindo para preencher e executa `st.stop()` — nenhuma aba é
renderizada até haver um nome preenchido.

## 2. Aba "Abrir chamado"

### 2.1 Formulário (`st.form`, `clear_on_submit=True`)

| Campo | Tipo | Opções / Regras |
|---|---|---|
| Módulo SAP | `st.selectbox` | `FI, CO, MM, SD, PP, QM, PM, HCM, BASIS, ABAP, Outro` |
| Prioridade | `st.selectbox` | `Baixa, Média, Alta, Crítica`, default = `Média` (index=1) |
| Descrição do problema | `st.text_area` (altura 160) | texto livre, obrigatório |
| Checkbox "Tentar resolução automática com IA agora" | `st.checkbox` | default = `True` |
| Botão | `st.form_submit_button("Abrir chamado")` | |

### 2.2 Comportamento ao submeter

1. Chama `backend.criar_chamado(usuario=usuario_atual, modulo_sap, descricao, prioridade)`
   dentro de um `try/except`.
2. Se der erro (`ValueError`, ex. campo vazio): `st.error(str(erro))`.
3. Se der certo: `st.success(f"Chamado #{ticket_id} aberto com sucesso.")`.
4. **[NOVO R4]** Valida a prioridade:
   - Chama `aiengine.sugerir_prioridade(descricao, modulo_sap, prioridade_cliente)`
   - Se houver divergência: mostra `st.warning()` alertando cliente da sugestão da IA
   - Notifica equipe automaticamente
5. Se o checkbox de IA estiver marcado:
   - Mostra `st.spinner("Analisando chamado...")` enquanto chama
     `aiengine.resolver_ticket_com_ia(ticket_id)`.
   - **[MODIFICADO R5]** Atualiza status para `"Aguardando confirmação"`
   - Grava a solução sugerida via `backend.resolver_chamado()`
   - Mostra a solução retornada em `st.info(solucao)`.
   - **[NOVO R5]** Exibe **dois botões de confirmação**:
     - `"✓ Resolveu meu problema!"` → chama `aiengine.aplicar_sugestao_ia(confirmou=True)` → fecha como `Resolvido`
     - `"✗ Não resolveu, quero falar com a equipe"` → chama `aiengine.aplicar_sugestao_ia(confirmou=False)` → volta para `Aberto`
   - Origem da solução exibida (KB, similar ou IA)
6. Se o checkbox estiver desmarcado: `st.caption` apontando para a aba
   "Meus chamados".

## 3. Aba "Meus chamados"

### 3.1 Controles superiores

| Elemento | Tipo | Comportamento |
|---|---|---|
| Filtro de status | `st.selectbox` | Opções: `Todos, Aberto, Em Atendimento, Resolvido` |
| Botão "🔄 Atualizar" | `st.button` | Chama `st.rerun()` |

### 3.2 Auto-refresh
Se a biblioteca `streamlit_autorefresh` estiver instalada
(`HAS_AUTOREFRESH = True`), chama `st_autorefresh(interval=15000, key="refresh_meus_chamados")`
— a aba se atualiza sozinha a cada 15 segundos. Se a lib não estiver
instalada, a tela funciona normalmente, só sem o auto-refresh (o botão
manual cobre o caso).

### 3.3 Tabela de chamados
Fonte: `backend.listar_chamados_por_usuario(usuario_atual, filtro)`.

Se vazio: `st.info("Você ainda não tem chamados nessa situação.")`.

Se houver dados, renderiza `st.dataframe` com as colunas, nesta ordem
(apenas as que existirem no resultado):
`id, modulo_sap, prioridade, status, data_criacao, prazo_limite`.
(`solucao_aplicada` **não** aparece na tabela — só no detalhe, para não
poluir a listagem com textos longos.)

### 3.4 Detalhe do chamado selecionado

Abaixo da tabela (sem `st.expander` — direto na tela):
- `st.selectbox` para escolher o `id` entre os disponíveis na tabela
  filtrada.
- Busca o registro completo via `backend.obter_chamado(id)`.
- Exibe:
  - `**Status:**` valor do status.
  - `**Descrição:**` descrição original do problema.
  - Se houver `solucao_aplicada`: bloco `**Solução aplicada:**` seguido de
    `st.info(solucao)` e `st.caption` com a origem (`IA`, `Base de
    Conhecimento` ou `Manual`).
  - Se não houver solução ainda: `st.caption("Ainda sem solução registrada — aguardando atendimento.")`.

#### 3.4.1 Seção de Conversa **[NOVO R7]**

Após a solução, exibe uma seção de **comentários/interações**:

- **Listagem de mensagens**: todas as interações do chamado, cada uma em um container com border.
  - Cada mensagem mostra: emoji/role (👤 Cliente / 👨‍💼 Equipe), username, data, e conteúdo.
  - Ordenadas por data crescente (mais antigas primeiro).
  - Se vazio: `st.info("Nenhuma mensagem ainda.")`.

- **Formulário de nova mensagem**: `st.form` com:
  - `st.text_area` para digitar (altura 80, placeholder "Digite aqui...")
  - Botão "Enviar mensagem".
  - Ao submeter: chama `backend.criar_interacao(ticket_id, usuario_id, mensagem)`.
  - Sucesso: `st.success` + `st.rerun()`.
  - Erro: `st.error(str(erro))`.

## 4. Estados a cobrir (checklist para quem for implementar/revisar)

- [ ] Sidebar sem nome preenchido → tela bloqueada com aviso.
- [ ] Formulário de abertura com campo obrigatório vazio → erro amigável,
      formulário não limpa (o usuário não perde o que digitou nos outros
      campos — comportamento padrão do Streamlit quando a validação falha
      no backend, já que `clear_on_submit` só limpa em caso de sucesso real
      do `st.form_submit_button`, e aqui a limpeza é condicionada ao fluxo
      do `try`).
- [ ] Abertura de chamado com IA acionada e API key ausente/inválida → erro
      deve aparecer via `st.error`, sem quebrar a tela.
- [ ] "Meus chamados" vazio (usuário novo) → mensagem amigável, não erro.
- [ ] Troca de filtro de status → tabela atualiza sem duplicar chamados.
- [ ] Auto-refresh não deve resetar o texto que o usuário está digitando no
      formulário de abertura (cuidado ao portar essa lógica: o
      `st_autorefresh` está isolado na aba "Meus chamados", não deve ser
      colocado no nível da página inteira).

## 5. Fora de escopo desta tela (propositalmente)
- Login real / senha.
- Edição ou cancelamento de um chamado já aberto pelo próprio cliente.
- Anexar arquivos/prints ao chamado.
- Avaliação de satisfação após a resolução.
