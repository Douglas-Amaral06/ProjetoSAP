# 00 — Visão Geral do Projeto

## Nome
Helpdesk SAP com IA (piloto interno)

## Objetivo
Sistema interno inspirado no Fluig para abertura e resolução de chamados de
suporte SAP, com um diferencial: **uma camada de IA que analisa o chamado,
sugere solução com base em normas internas cadastradas, e reaproveita
soluções já dadas para problemas repetidos** (base de conhecimento).

Este é um **piloto**: o objetivo não é entregar um produto completo de
primeira, e sim validar o fluxo (cliente abre → IA ou equipe resolve →
solução fica registrada) com o menor atrito possível.

## Público / Perfis de uso

| Perfil | O que faz | Interface |
|---|---|---|
| Cliente (usuário SAP) | Abre chamados, acompanha status, vê a solução quando sai | `cliente_app.py` |
| Equipe SAP | Gerencia a fila, resolve manualmente ou aciona a IA, mantém normas e base de conhecimento, acompanha SLA | `equipe_app.py` |

As duas interfaces são **aplicações Streamlit separadas**, cada uma rodando
em uma porta local própria, mas compartilhando o mesmo banco SQLite através
de um módulo `backend.py` comum. Não há um servidor/API intermediário nesta
fase — a sincronização é feita diretamente via arquivo `.db` compartilhado.

## Stack tecnológico

| Camada | Tecnologia |
|---|---|
| Linguagem | Python 3.11+ |
| Interface web | Streamlit |
| Dados / dashboards | Pandas |
| Orquestração de IA | LangChain |
| Modelo de IA | Google Gemini (`gemini-2.5-flash`) via API |
| Banco de dados | SQLite3 (arquivo local, modo WAL) |
| Tema visual | Dark, inspirado no GitHub Dark |

> **Nota sobre Databricks**: o stack originalmente citava Databricks para
> "treinar o modelo". Como a IA usada é o Gemini via API (modelo de terceiros,
> não treinado por nós), não há etapa de treinamento neste piloto. O
> "aprendizado" do sistema hoje é a base de conhecimento (tabela
> `knowledge_base`), que registra e reaproveita soluções. Databricks só
> voltaria a fazer sentido se, no futuro, o projeto migrar para um modelo
> próprio fine-tunado com o histórico real de chamados. Ver `05-roadmap-melhorias.md`.

## Arquitetura (alto nível)

```
┌─────────────────────┐        ┌─────────────────────┐
│   cliente_app.py     │        │    equipe_app.py     │
│  (porta 8501)         │        │   (porta 8502)         │
└──────────┬───────────┘        └──────────┬───────────┘
           │                                │
           │        importam mesmo módulo   │
           └───────────────┬────────────────┘
                            │
                    ┌───────▼────────┐
                    │   backend.py    │  ← regras de negócio + acesso a dados
                    └───────┬────────┘
                            │
                    ┌───────▼────────┐
                    │ sap_chamados.db │  (SQLite, modo WAL)
                    └────────────────┘
                            ▲
                            │
                    ┌───────┴────────┐
                    │  aiengine.py    │  ← integração com Gemini via LangChain
                    └────────────────┘
```

## Estrutura de arquivos do projeto

```
helpdesk_sap_ia/
├── backend.py          # regras de negócio + acesso ao SQLite (compartilhado)
├── aiengine.py          # integração com Gemini via LangChain (compartilhado)
├── theme.py              # CSS do tema dark (compartilhado)
├── cliente_app.py       # interface de quem ABRE o chamado
├── equipe_app.py         # interface de quem RESOLVE o chamado
├── requirements.txt
├── .env.example
├── sap_chamados.db       # criado automaticamente na 1ª execução
└── docs/                 # esta pasta de documentação
```
